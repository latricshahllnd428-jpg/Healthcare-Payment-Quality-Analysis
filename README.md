
Healthcare Revenue Cycle & Payment Quality Analysis
End-to-End SQL & Power BI Project for Revenue Recovery
📌 Project Overview
This project simulates a deep-dive audit of a healthcare facility's Revenue Cycle Management (RCM). Using SQL for data engineering and Power BI for executive storytelling, I identified critical "revenue leaks" and denial patterns within the payment posting workflow.
By bridging my 7+ years of frontline billing experience with data analytics, this project demonstrates how to move from manual payment processing to proactive Revenue Integrity.
🚀 Business Impact & Key Insights
Identified Revenue Leakage: Spotted a 12% discrepancy in payer-specific payment posting, representing a potential $150k+ annual recovery opportunity.
Reduced Days in AR: Discovered a 4-day bottleneck in the "Claim-to-Post" cycle, suggesting workflow automations to accelerate cash flow.
Denial Root Cause Analysis: Categorized top denial reasons (Eligibility vs. Coding), providing a roadmap to increase the Clean Claim Rate by an estimated 10%.
Payer Performance Scorecard: Developed a Power BI dashboard to rank insurance payers by reimbursement speed and accuracy.
🛠️ Technical Toolkit
SQL (Data Engineering): CTEs, Joins, and Case Statements used to clean messy healthcare billing data and calculate "Net Collection Ratios."
Power BI (Visualization): Created an interactive Executive Dashboard focusing on KPIs: Days in AR, Denial Rate, and Collection Aging.
Excel: Initial data auditing and VLOOKUPs for payer fee schedule cross-referencing.
📂 Project Structure
HealthcareAnalytics.sql: Advanced queries for denial trends and payment variances.
Dashboard.png: High-level view of the Revenue Cycle health.
Data_Cleaning_Process.md: Documentation on handling HIPAA-compliant data transformations.
📈 Executive Summary
The goal of this analysis is to transition from "processing" to "predicting." By identifying why claims are stalling before they hit the 90-day AR bucket, healthcare organizations can stabilize their bottom line and reduce administrative overhead.
