# Healthcare-Payment-Quality-Analysis
End-to-end SQL, data cleaning, and Power BI dashboard project analyzing healthcare payment posting quality.

## Project Files
- SQL Query: HealthcareAnalytics.sql
- Cleaned Data: Healthcare_PaymentQuality_Analysis.xlsx
- Power BI Dashboard: Healthcare_Payment_Quality_Dashboard.pbix
- Dashboard Screenshot: Dashboard.png

## Project Overview
This project analyzes healthcare payment posting quality using three core steps:
1. Writing SQL queries to extract and prepare the raw healthcare payment data.
2. Cleaning and transforming the data in Excel for usability.
3. Building an interactive Power BI dashboard to evaluate payment posting performance by payer and billing system.

The objective is to identify trends, posting problems, and areas for process improvement.

## Tools Used
- SQL
- Excel
- Power BI
- GitHub

## Key Insights
- Identified payers with higher average percent of posting issues.
- Compared total deposits and problem amounts across payers.
- Analyzed posting quality by billing system (E360, Epic, EyecareBillingSys, NextGen, USOncologySys).
- Highlighted trends that can help improve operational efficiency and accuracy.

## How to Use This Repository
1. Review the SQL script to understand the data extraction and preparation steps.
2. Open the Excel file to see the cleaned dataset.
3. Open the Power BI file to explore the interactive dashboard.
4. View the dashboard screenshot for a quick visual reference.
